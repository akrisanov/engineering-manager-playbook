# Разработчик

| **Грейд**                                 | Middle                                                                                               |
| ----------------------------------------- | ---------------------------------------------------------------------------------------------------- |
| **Для повышения необходимо**              | Показать достаточный уровень автономности и стабильно решать поставленные задачи                     |
| **Основная цель**                         | Научиться приносить пользу компании _самостоятельно_                                                 |
| **Умение планировать свою работу на**     | 1 – 2 недели                                                                                         |
| **Решаемые задачи**                       | Четко определены, но могут включать моменты для проработки                                           |
| **Сложность**                             | Требуется коммуникация с командой для формирования оптимального решения                              |
| **Управление рисками**                    | Обращение к менеджеру или тимлиду заранее, до момента появления блокеров                             |
| **Необходимость ревью выполненных задач** | Да – коллегами и/или руководителем команды                                                           |
| **Экспертность в домене**                 | Компетентность в своем домене. Знакомство с другими доменами в вашей команде.                        |
| **Понимание продукта**                    | На уровне всей функциональности, которую разрабатывает команда                                       |
| **Влияние на людей**                      | Только на себя                                                                                       |
| **Коммуникация**                          | Умение рассказать о своем прогрессе и достигнутых результатах, а также возможных рисках              |
| **Вклад в команду**                       | Открытость к обсуждениям, активное в них участие и привнесение идей по улучшению процессов и практик |
| **Оценка перформанса**                    | Руководитель команды + лид направления                                                               |
| **Опыт продуктовой разработки**           | 2 – 4 года                                                                                           |
