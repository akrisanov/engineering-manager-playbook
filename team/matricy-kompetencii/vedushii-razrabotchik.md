# Ведущий разработчик

| **Грейд**                                | Staff                                                                                                                                                                                                                                                                                                  |
| ---------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Для повышения необходимо**             | <p>Показать навык определять <strong>стратегию</strong> для достижения целей сервиса / продукта / компании.</p><p>Уметь ставить цели в рамках этой стратегии команде(ам).</p>                                                                                                                          |
| **Основная цель**                        | Помочь бизнесу кратно расти говоря что нужно для этого делать                                                                                                                                                                                                                                          |
| **Умение планировать свою работу на**    | 2-3 месяца                                                                                                                                                                                                                                                                                             |
| **Решаемые задачи**                      | <p>Включают необходимость проведения исследований со значительной долей неопределенности.</p><p>Обязательно включают этапы проектирования и обсуждения с командой или тех.менеджером.</p><p>Влияют на всю команду и/или продукт.</p><p>Большая часть декомпозированных задач делегируется команде.</p> |
| **Сложность**                            | <p>Сложные и комплексные задачи с множеством компонентов.</p><p>Требуют декомпозиции, планирования и вовлечения команды для достижения общих целей.</p>                                                                                                                                                |
| **Управление рисками**                   | Умение минимизировать технические риски в своей работе, а также в работе команды, в том числе за счет эффективной коммуникации                                                                                                                                                                         |
| **Необходимость ревью выполнения задач** | Нет, в редких случаях CTO                                                                                                                                                                                                                                                                              |
| **Экспертность в домене**                | Полное понимание своего домена, экспертность в смежных: бизнес, архитектура                                                                                                                                                                                                                            |
| **Понимание продукта**                   | На уровне всей функциональности, которую разрабатывает команда                                                                                                                                                                                                                                         |
| **Влияние на людей**                     | Команда или несколько взаимодействующих команд                                                                                                                                                                                                                                                         |
| **Коммуникация**                         | Выстраивание прозрачных и доверительных отношений с командой и менеджментом                                                                                                                                                                                                                            |
| **Вклад в команду**                      | Техническое лидерство, умение демонстрировать достижения команд(ы), найм                                                                                                                                                                                                                               |
| **Оценка перформанса**                   | Тимлид / Head of Engineering                                                                                                                                                                                                                                                                           |
| **Опыт продуктовой разработки**          | > 7 – 10 лет                                                                                                                                                                                                                                                                                           |
