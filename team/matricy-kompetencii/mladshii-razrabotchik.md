# Младший разработчик

| **Грейд**                                 | Junior                                                                                                                                                     |
| ----------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Основная цель**                         | Научиться приносить пользу компании с поддержкой команды или ментора                                                                                       |
| **Умение планировать свою работу на**     | 1 – 3 дня                                                                                                                                                  |
| **Решаемые задачи**                       | Четко определены, уровень неопределенности минимальный                                                                                                     |
| **Сложность**                             | Требуется минимальное взаимодействие с командой                                                                                                            |
| **Управление рисками**                    | Обращение к менеджеру или тимлиду при возникновении сложностей                                                                                             |
| **Необходимость ревью выполненных задач** | Да – коллегами и/или руководителем команды                                                                                                                 |
| **Экспертность в домене**                 | Низкая, продолжает изучать домен (например, фин.учет) и принятые в команде инженерные практики                                                             |
| **Понимание продукта**                    | На уровне функциональности, с которой сталкивается в ежедневной работе                                                                                     |
| **Влияние на людей**                      | Только на себя                                                                                                                                             |
| **Коммуникация**                          | Умение задавать вопросы [правильно](https://hbr.org/2018/05/the-surprising-power-of-questions), а также рассказать о своем прогрессе в работе над задачами |
| **Вклад в команду**                       | Изучение и следование принятым практикам, участие в обсуждениях (хотя бы на уровне вопросов)                                                               |
| **Оценка перформанса**                    | Руководитель команды + лид направления                                                                                                                     |
| **Опыт продуктовой разработки**           | 0 – 2 года                                                                                                                                                 |
